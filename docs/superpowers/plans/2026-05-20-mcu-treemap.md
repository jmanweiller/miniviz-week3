# MCU Treemap Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a single-page Power BI report (PBIP) featuring a native treemap of MCU films sized by box office and colored by RT critic score, with a poster-driven tooltip page.

**Architecture:** A PBIP project with a single import-mode semantic model from one CSV, one report page hosting the treemap and supporting visuals, and a hidden report-page tooltip showing the hovered film's poster and stats. Posters hosted as JPEGs in this repo on GitHub and referenced via `raw.githubusercontent.com` URLs through a column marked as Image URL.

**Tech Stack:** Power BI Desktop, PBIP format, TMDL semantic model, PBIR report format, native treemap visual. Data sourced via Wikipedia/Box Office Mojo/Rotten Tomatoes.

## Task 1: Compile MCU dataset

**Files:**
- Create: `Data/mcu_films.csv`

- [ ] **Step 1: Compile film list and stats**

Compile one row per theatrical MCU film from Iron Man (2008) through the most recent release as of 2026-05-20. Use Wikipedia "List of Marvel Cinematic Universe films" as the spine; cross-check Box Office Mojo for grosses and Rotten Tomatoes for scores.

Columns (exact header names, comma-separated, UTF-8, no BOM):
```
Title,Phase,ReleaseDate,WorldwideBoxOffice,Budget,RTCriticScore,RTAudienceScore,Director,RuntimeMinutes
```

- `Phase` formatted as `"Phase 1"`, `"Phase 2"`, etc.
- `ReleaseDate` in ISO `YYYY-MM-DD`
- `WorldwideBoxOffice` and `Budget` as raw USD integers (no `$` or commas)
- `RTCriticScore` and `RTAudienceScore` as integers 0–100
- `Director` as a single string; for multi-director films use `"Russo Brothers"` etc.
- `RuntimeMinutes` as integer

- [ ] **Step 2: Validate the CSV**

Run: `Import-Csv "Data/mcu_films.csv" | Measure-Object | Select-Object -ExpandProperty Count`
Expected: roughly 33–37 rows. Each row must have all 9 columns populated.

Run: `Import-Csv "Data/mcu_films.csv" | Where-Object { -not $_.WorldwideBoxOffice -or -not $_.RTCriticScore }`
Expected: no output (no rows with missing critical fields).

    $_ | Add-Member -NotePropertyName PosterUrl -NotePropertyValue "https://raw.githubusercontent.com/$owner/miniviz-week3/$branch/posters/$slug.jpg"

$films | Export-Csv "Data/mcu_films.csv" -NoTypeInformation -Encoding UTF8

## Notes & risks

- **Wikipedia poster licensing:** Posters are non-free fair-use images. Hosting them on a public GitHub repo for personal/contest use is the standard miniviz practice but not legally clean. If this matters, swap to officially licensed press images or use color-blocked tiles as fallbacks.
- **Absolute CSV path:** Task 5 leaves the absolute path in the M query for local dev. If you publish to the Service or hand the PBIP to someone else, parameterize the path first.
- **Tooltip poster sizing:** Wikipedia thumbnails vary in aspect ratio. The Image visual will preserve aspect; expect some letterboxing in the tooltip.
- **2025–2026 films:** Confirm any 2026 MCU releases that may be eligible by 2026-05-20 are included or explicitly excluded. The dataset is frozen at build time.
